# Brainstorm — /superpowers-brainstorm

Before implementing any feature or architectural decision, run this workflow:

1. **Problem framing**: Restate the goal in one sentence. What problem does this solve?
2. **3 approaches**: Generate at least 3 distinct implementation approaches.
   For each: implementation sketch, trade-offs, risks, maintenance burden.
3. **Constraints check**: What are the hard constraints (performance, security, backwards-compat)?
4. **Recommendation**: Pick the approach that best fits the constraints. Justify in 2 sentences.
5. **Open questions**: List anything that needs user input before proceeding.

Output format: numbered list per approach, then recommendation + open questions.
Do NOT start writing code until this workflow is complete and the user has confirmed the approach.

---

> This command is the portable equivalent of `superpowers:brainstorming` (L2 mandatory before
> features). It works in all environments — with or without the superpowers plugin installed.
