# Research Notes

`docs/research/` is for source collection and investigation notes before a decision becomes canonical.

## Ownership boundary

- Research is **not policy**.
- Promote accepted decisions to `architecture-decisions/`, `core/`, `docs/operations/`, or `patterns/`.
- Do not cite research as the final rule when a canonical owner exists.

## Use when

- Comparing tools/services before adding an external system.
- Collecting official docs and examples before creating an ADR.
- Recording exploratory findings that are not yet stable enough for policy.
